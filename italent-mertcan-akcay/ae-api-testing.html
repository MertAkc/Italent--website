Seminarie AE: API

Bij het begin van het seminarie stelden de werknemers van AE zichzelf in het kort voor. Het bedrijf is opgericht in 1999 en zijn gevestigd in Leuven. Kristof van Kriekingen, Quality Assurance consultant die al 6 jaar bij AE werkt en zijn collega Dries Wouters, ook een Quality Assurance consultant die bij AE 3,5 jaar ter dienst staat. Vervolgens werd er in het kort de visie van AE uitgelegd waarbij het bedrijf streeft naar nummer één om problemen van klanten op te lossen. AE doet dit door een unieke strategie toe te passen namelijk co-think; co-act; en co-create;

Na de voorstelling begonnen de presentators over Apis (Application Programming Interfaces). Apis zijn interfaces die met elkaar kunnen communiceren in een netwerk. Belangrijke begrippen zoals API-verzoeken, endpoints, HTTP-methoden, HTTP-headers en API-responsen werden aan de studenten uitgelegd. De presentators legden de nadruk op dat deze concepten zeer belangrijk waren bij softwareontwikkeling en softwaretesting in het bedrijfsleven.

Vervolgens legden de presentators uit hoe je een JSON-object kon meegeven aan een API-call en een JSON-object eruit kon halen van de response uit een API-call. Nadien werd er ook gesproken over de hoe je de status code 200, 400 en etc. eruit kon halen en die statusmeldingen kon weergeven.

De CRUD-operaties met verschillende methoden werden besproken. De CRUD staat voor Create Read Update Delete en het belang van zowel handmatige als geautomatiseerde tests werd benadrukt. Aan de hand van deze voorbeelden kregen studenten de uitleg wat deze operaties waren en het belang daarvan. De presentatie omvatte ook een praktische lab oefeningen waarbij de studenten Postman moesten gebruiken om API-calls te maken en deze te capteren op de browser. Door middel van deze lab sessies kregen studenten de kans om hun vaardigheden te verbeteren en hun begrip van belangrijke concepten te verdiepen. Studenten hadden de optie om vragen te stellen tijdens het seminarie.

Het seminarie van AE was een boeiende en een leerrijke ervaring. Ik had al een basiskennis van API maar in dit seminarie hebben de presentators de API in detail vertelt. Zo kon ik mijn kennis van API verdiepen en de basis weer opfrissen. De labs waren zeer interessant en ook moeilijk maar de presentators deden actief mee om studenten te helpen. Mijn sterke punten waren het uitvoeren van een “GET” en een “POST” call. Dit was dankzij het vorige seminarie. Mijn zwakke punten waren de “UPDATE” en “DELETE”. De presentators waren behulpzaam en gaven studenten persoonlijke advies om de API’s uit te kunnen werken.

Dit seminarie is ook gerelateerd in de IT-sector. Tegenwoordig gebruiken bedrijven meer en meer API in hun applicaties waardoor het eigenlijk een onmisbaar onderwerp is geworden in de sector. Dus ik ben blij en fier op mezelf dat ik dit seminarie kon deelnemen. De kennis hierover zal ik zeker mee kunnen nemen naar het echte bedrijfsleven. Ik heb deze activiteit ook meegenomen in de portfolio om de nadruk op API te leggen en ook mijn groei in het creëren van API-calls voor te kunnen leggen
